# Issue Templates

Use these condensed templates when the user did not provide an issue.

Keep issue titles short and action-oriented. Fill only the parts supported by the staged diff and the user prompt. Do not invent detailed acceptance criteria if the evidence is weak.

## Feature

Title style:

- `Create ...`
- `Add ...`
- `Support ...`

Body:

- Problem / motivation
- Proposed solution
- User story
- Acceptance criteria
- Non-goals
- Technical notes

## Bug

Title style:

- `Fix ...`
- `Resolve ...`
- `Correct ...`

Body:

- Summary
- Reproduction
- Expected behavior
- Actual behavior
- Suspected cause
- Acceptance criteria

## Chore

Title style:

- `Refactor ...`
- `Update ...`
- `Improve ...`
- `Clean up ...`

Body:

- Summary
- Scope
- Constraints
- Plan
- Risks
- Acceptance criteria
